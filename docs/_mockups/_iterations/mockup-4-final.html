02 -- CAD Architecture

CATIA V6 Parametric Design

~60-80 hours learning CATIA V6 from scratch. The entire assembly is skeleton-driven: a master skeleton holds all parameters, and parts read from it through formulas -- never referencing each other directly.

[ CATIA: full assembly cross-section ]
Assembly

Skeleton-Driven Architecture

Single source of truth. All positioning in skeleton -- parts never reference each other. Change one value, everything updates.

Click for principles
  • No B-Rep references -- edges/faces break on feature reorder
  • Body-level patterning -- more stable than feature-level
  • Placeholder gears -- define interface, swap in real teeth later
  • Chain-of-zeros -- zero offsets everywhere eliminates drift

03 -- DFM & Tolerancing

FDM Calibration Pipeline

ISO fit designations describe intent. FDM offsets approximate those fits. Calibration coupons validate every bearing interface before the full build.

Design Decision

All bearing interfaces target finger-press transition fits. FDM surface texture provides grip, so press fits are unnecessary for a backdrivable actuator.

InterfaceOffsetStatus
Main bearing bore+0.10mmValidated
Carrier bearing bore+0.10mmApplied
Lid bearing bore+0.10mmValidated
Planet bearing bore+0.15mmPending
Ring-to-housing+0.30mmApplied

Self-tapping validated: M3=2.6mm, M4=3.6mm, M5=4.6mm pilots. Simpler than heat inserts for prototyping.

05 -- Testing & Validation

Structured Root Cause Analysis

5 structured tests with an unknowns register, dependency mapping, and requirements traceability. Each test defines the question, measurement, and acceptance criteria before testing begins.

T-001

Gear Mesh Feel

Root Cause Found Herringbone misalignment -- at least one planet installed with incorrect helix orientation.

Click for analysis

Question: Why does the gearbox feel notchy?

Test: Carrier + planets in ring (no sun) = smooth. Sun inserted = clicky. Backlash present.

Root cause: Herringbone helix angle too subtle to identify during assembly. Easy to install a planet backwards.

Decision: Switch to spur gears for Rev 00B. Herringbone over-constrains mesh at this scale with no measurable benefit.

T-002

Lid Drag

Root Cause Found Lid double-constrains ring gear + carrier. Tolerance stackup pushes carrier higher than nominal.

Click for analysis

Root cause: Lid simultaneously clamps ring gear to housing AND constrains carrier top via bearing recess. Two independent constraint paths conflict.

Decision: Decouple constraints -- add axial clearance on carrier top shaft.

T-003

Shoulder Deformation

Root Cause Found Print artifacts on top carrier bearing shoulders due to orientation trade-off.

Click for analysis

Finding: Printing output-shaft-up gives good top bearing surface but planet bearing shoulders have overhang artifacts.

Decision: Reverse print strategy -- top carrier loses walls, bottom gets all walls. Eliminates overhang on critical surfaces.

T-004

Carrier Indexing

No Issue Load path analysis shows no operational force drives carrier clocking.

Click for analysis

Finding: ~0.5mm play exists, but planets push both carrier halves in the same direction via shared pins. No opposed force to drive rotation.

Decision: No design change needed. Optional indexing for assembly repeatability only.

T-005

Bolt Torque Sensitivity

Marginal Narrow window between carrier play and planet binding. Confirms shoulder design sensitivity.

Click for analysis

Finding: At 1.1 Nm: noticeable drag. Backed off 1 turn: all spin freely. Re-torqued incrementally: 1/3 free, 2/3 slight drag.

Decision: Confirms T-003 findings. Shoulder redesign required for Rev 00B.

Methodology

Each unknown scored by risk x uncertainty. Tests resolve bottom-up: component, interface, subsystem, system. All results trace back to requirements (R-01 through R-11).
