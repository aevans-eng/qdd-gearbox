# STEP Output — Aaron's QDD Gearbox

Generated by `generate_step_aaron.py`. These are the actual gear body solids for import into CATIA.

## Pipeline Overview

`generate_step_aaron.py` does the following:

1. **Define gear params** — R48/P18/S12, 3 planets, 75mm ring inner diameter, 18.6mm face width, 30° helix angle
2. **Validate config** — checks mesh constraint ($R = S + 2P$), assembly constraint ($(R+S) \div N$ is integer), and planet non-interference
3. **Generate 2D tooth profiles** — pygeartrain creates cycloidal (epi/hypocycloid) tooth curves, NOT involute. The `b=0.5` parameter controls the epi/hypo mix ratio.
4. **Scale to physical size** — everything is scaled so the ring gear's maximum radius = 37.5mm (i.e. ring inner diameter = 75mm, matching the CATIA skeleton)
5. **Apply profile offset** — each gear gets -0.025mm radial offset (thins teeth), giving ~0.05mm clearance at each mesh interface
6. **Build 3D herringbone solids** — lofts twisted 2D profiles through intermediate cross-sections per half (controlled by `LOFT_SECTIONS_PER_HALF`). The two halves have opposite twist directions, meeting at $z=0$ to form the herringbone V-shape.
7. **Position planets** — planet $i$ is translated to $(r \cos\theta_i,\; r \sin\theta_i)$ where $r$ = carrier radius and $\theta_i = 120° \times i$
8. **Build ring halves** — outer cylinder ($r = 47.5$ mm) minus internal tooth loft, split at $z=0$ for herringbone assembly
9. **Export** — two types: `_PRINT` files (individual parts for manufacturing) and `_CAD` file (multi-body compound for CATIA import as a part)

## Key Concepts

### Why cycloidal, not involute?
The pygeartrain library uses epi/hypocycloidal tooth profiles. The skeleton param `gear_module = 2.5mm` does not apply to these gears — module is an involute concept. The ring inner diameter (75mm) is what ties the STEP geometry to the CATIA skeleton.

### Planet mesh rotation
Each planet must be rotated so its teeth mesh with the sun and ring at its orbital position:
$$w = \left(1 - \frac{R}{P}\right) \times \theta_i$$
where $\theta_i$ is the planet's angular position (0°, 120°, 240°). This ensures correct tooth engagement at every orbital location.

### Ring compensation (0.2°)
The ring gear needs a small angular compensation because planets and ring rotate around different centers. At $z \neq 0$, the helical twist creates angular drift between the planet-ring mesh point (which rotates around the planet center) and the ring profile (which rotates around the global origin). The compensation formula:
$$\Delta\alpha = z \times \tan(\text{helix}) \times \left(\frac{1}{r_{\text{ring,inner}}} - \frac{1}{r_{\text{mesh}}}\right)$$
The script uses a validated empirical value of 0.2°. See `Herringbone Planetary Pipeline.md` for the full derivation and iteration history.

### Herringbone construction
Each gear is built from 3 profiles lofted together:
- **Bottom** ($z = -9.3$ mm): profile twisted by helix angle
- **Mid** ($z = 0$): untwisted base profile
- **Top** ($z = +9.3$ mm): profile twisted in the opposite direction

Each half is a ruled surface (straight-line loft between 2 profiles). The helix angle is correctly applied to the endpoint rotations, but intermediate geometry takes a linear shortcut rather than a true helical path. Good enough for prototyping; improving helix fidelity would need a sweep-along-path approach (attempted loft intermediate sections on 2026-03-01, caused geometry issues — see changelog).

- **Bottom half** ($z = -9.3$ to $0$ mm): profile progressively twisted by helix angle
- **Top half** ($z = 0$ to $+9.3$ mm): profile twisted in the opposite direction
- **Midplane** ($z = 0$): untwisted base profile, shared by both halves

The two halves are fused at $z=0$ to form the V-shaped herringbone teeth.

### Coordinate system
- **Origin**: center of sun and ring gears
- **Z-axis**: gear rotation axis, $z=0$ is the gear midplane (matches skeleton `Gear_midplane`)
- **Planets**: translated to their orbital positions (not at origin)
- All dimensions in mm

## Output Files

### Manufacturing (`_PRINT`) — clean individual parts for slicer/manufacturer

| File | Description |
|------|-------------|
| `sun_PRINT.step/.stl` | Sun gear (12 teeth), centered at origin |
| `planet_0_PRINT.step/.stl` | Planet 0 (18 teeth), at 0° orbital position |
| `planet_1_PRINT.step/.stl` | Planet 1 (18 teeth), at 120° orbital position |
| `planet_2_PRINT.step/.stl` | Planet 2 (18 teeth), at 240° orbital position |
| `ring_bottom_PRINT.step/.stl` | Ring bottom half (48 teeth), $z = -9.3$ to $0$ mm |
| `ring_top_PRINT.step/.stl` | Ring top half (48 teeth), $z = 0$ to $+9.3$ mm |

### CAD Integration (`_CAD`) — multi-body part for CATIA

| File | Description |
|------|-------------|
| `gearbox_CAD.step` | All gears in one compound — import as **Part** in CATIA, not assembly |

The `_CAD` file is a `TopoDS_Compound`: all bodies share one coordinate system, nothing can drift. Import as a part, save as native CATPart, then constrain to skeleton. See `docs/catia-step-integration.md` for full workflow.

## STEP Limitations

STEP files don't carry per-body coordinate systems or named datums. Every body's embedded planes/axes land at the global origin, even if the body itself is translated (e.g. planets). This is a format limitation — AP203, AP214, and AP242 all have this problem for CadQuery-generated geometry.

**After importing into CATIA, create reference geometry from pickable faces:**
- Sun's central bore at origin → extract axis
- Ring half's flat split face at $z=0$ → midplane
- Planet\_0 center at $(r_{\text{carrier}}, 0, 0)$ on +X axis → rotation reference

## Regenerating

Requires the `pygeartrain` conda environment (cadquery + OCP dependencies):

```
conda activate gear_design
cd pygeartrain/
python generate_step_aaron.py
```

The script prints computed dimensions — use these to update CATIA skeleton params if tooth counts or scaling change.

## Relation to Previous Pipeline

The older pipeline (`generate_planetary_cad_v16.py`) exported XYZ point files for manual loft construction in CATIA via a VBA macro. This script replaces that workflow by exporting finished STEP solids directly — no macro, no manual lofts, no spline stitching.

| | Old (v16) | New (generate_step_aaron.py) |
|---|---|---|
| Tooth counts | R30/P11/S8 | R48/P18/S12 |
| Output format | XYZ point files | STEP/STL solids |
| CATIA import | VBA macro + manual lofts | Direct STEP import |
| Ring construction | Manual Boolean in CATIA | Ring halves pre-built |
| Planet positioning | Pre-positioned in XYZ | Pre-positioned in STEP |
